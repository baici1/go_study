发现仅仅是`=`和`:=`的一个小小区别

>原因就是
>当我们在使用 := 时 会创建一个新的db变量,新的db会把全局变量db覆盖掉

其实很好理解.后续的代码我们都将使用db变量里面的函数,但是当你运行上面一段代码(有`:=`),就把全局的db变量覆盖,导致后续用的是覆盖之后的db变量
